## Featured Projects

### Mini CMS
A lightweight multilingual Laravel CMS for small websites, portfolios, business websites, blogs, and shared-hosting environments.

**Problem:** small clients and operators often need a practical CMS that deploys easily on shared hosting or simple VPS environments.

**Solution:** a Laravel/Filament CMS with SQLite support, multilingual content, RTL support, installer flow, deployment docs, release workflow, and production checklist.

**Solution:** a practical PHP/MySQL system with admin roles, event links, QR cards, check-in/check-out flows, exports, and operational logging.

## Production Questions I Ask Before Shipping

These are the five questions I use to evaluate whether a project is ready to move from idea/MVP to production.

| Question | Why it matters | Current answer across my work |
| --- | --- | --- |
| 1. What exact problem does it solve? | Prevents building vague tools with no buyer/user | Each serious repo is now mapped to a clear operational problem |
| 2. Who is the first real user? | Forces product focus and reduces feature drift | Operators, small businesses, hosting admins, event teams, or service teams |
| 3. Can it be deployed simply? | A product that cannot be installed reliably is not production-ready | Mini CMS, DPVHUB, VoIP, and Marshal BMS include deployment direction; others need cleanup |
| 4. What can break in real use? | Production failures usually come from auth, data, uploads, network, or unclear roles | Active projects are being documented around risk, roles, data flow, and deployment assumptions |
| 5. What is the next measurable milestone? | Prevents endless building without shipping | Public release, screenshots, demo, QA checklist, and first external users for Mini CMS |
